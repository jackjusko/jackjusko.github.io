Feature development and documentation:
In the root directory of this project, you have access to a markdown file named `analysis.md`. This file contains the detailed analysis of the project requirements, including the features, functionality, and constraints. Before starting any change or implementing any feature, please review the `analysis.md` file deeply to ensure that you are aligning with the project requirements. Any change you make or feature you implement should be based on the analysis provided in the `analysis.md` file.

For any change you're making to the project, whether it's a new feature or a bug fix, please ensure that you update the `analysis.md` file accordingly, with detailed documentation. This will help maintain a clear understanding of the project requirements and ensure that any changes are properly documented. Further, if you're adding a new feature, please include a detailed description of the feature, its functionality, and any constraints or assumptions. 

After implementing a change or a new feature, you must run two iterations of "feature analysis" - it is your job to review your own work critically, from the eyes of a manager who's considering potential bugs, issues, or weak points in the implementation. You think about edge cases, implementation and integration details, focusing strongly on a systems level evaluation that ensures no details has been missed in integrating your new feature seemlessly into the application. On the first iteration of this process, you write this analysis into a new markdown file with in the 'changes' folder in the root directory, which you name based on the task you're being asked to accomplish. You read this file and consider your analysis, and then proceed to implement all changes necessary to account for weak points in the analysis. Finally, you repeat this entire feature analysis process a second time: you review all your own work again, including the analysis you made in the first iteration, then you expand your analysis in your markdown file to cover anything new you have discovered in your considerations, and finally, you implement a new set of changes based on your final analysis document. Ensure that after your final set of changes, everything you've done for both the feature analysis and the overarching feature/change development has been documented and integrated into 'analysis.md'.
